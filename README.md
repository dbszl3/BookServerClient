# 📚 Book Rental Server
> **TCP 소켓 프로그래밍 기반 도서 대출 서버 프로젝트**

> 클라이언트가 서버에 접속하여 **회원가입, 로그인, 도서 조회, 도서 대출 및 반납** 기능을 이용할 수 있는 콘솔 기반 도서 대출 시스템입니다.
---
## 📖 프로젝트 개요
- TCP Socket Programming 기반 서버·클라이언트 시스템
- 다중 클라이언트 동시 접속 지원 (`fork()` 사용)
- 회원가입 및 로그인 기능 구현
- 도서 목록 조회 및 대출 / 반납 기능 제공
- 대출 기간(7일) 관리 및 연체 여부 판별
- 사용자 정보와 대출 기록을 파일로 저장하여 서버 종료 후에도 데이터 유지
- 콘솔 환경에서 서버와 클라이언트 간 상호작용
---
## ✨ 주요 기능
### 👤 회원 관리
- 회원가입 (`SIGNUP`)
- 로그인 (`LOGIN`)
- 사용자 정보 저장 및 인증

### 📚 도서 관리
- 도서 목록 조회 (`LIST`)
- 도서 대출 (`BORROW`)
- 도서 반납 (`RETURN`)
- 대출 가능 여부 표시

### ⏰ 대출 관리
- 기본 대출 기간 **7일**
- 대출 기간 초과 시 연체 여부 판별
- 반납 시 연체 메시지 출력

### 🔒 사용자 편의 기능
- 비밀번호 입력 시 `*` 표시 처리
- 서버 종료 후에도 회원 및 대출 정보 유지
---

## 💻 사용 명령어
| 명령어 | 설명 |
|---------|------|
| `SIGNUP` | 회원가입 |
| `LOGIN` | 로그인 |
| `LIST` | 도서 목록 조회 |
| `BORROW` | 도서 대출 |
| `RETURN` | 도서 반납 |
| `EXIT` | 프로그램 종료 |

## 📂 프로젝트 구조
```text
BookRentalServer
│
├── book_server.c      # 도서 대출 서버
├── book_client.c      # 클라이언트
├── books.txt          # 도서 목록 (최대 10권)
├── users.txt          # 회원 정보
├── borrow_log.txt     # 대출 기록
├── netprog2.h         # 공통 헤더
└── Makefile           # Build 및 Reset
```

## 📌 데이터 관리
- **users.txt**
  - 회원가입 시 아이디와 비밀번호 저장
  - 로그인 시 사용자 인증

- **books.txt**
  - 도서 목록 저장
  - 최대 10권 지원
  - 도서 대출 상태 관리

- **borrow_log.txt**
  - 대출 및 반납 기록 저장
  - 연체 여부 확인
---
## 🔄 Reset
```bash
make reset
```
실행 시
- 회원 정보 초기화
- 대출 기록 초기화
- 도서 상태 초기화 (모든 도서를 대출 가능 상태로 변경)
---
## 👩‍💻 개발 과정
- TCP 소켓 통신을 이용한 서버·클라이언트 구조를 구현
- `fork()`를 활용하여 다중 클라이언트 동시 접속을 처리
- 파일 입출력을 활용하여 회원 정보와 대출 기록을 영구적으로 저장
- 비밀번호 입력 시 `*`가 출력되도록 구현
